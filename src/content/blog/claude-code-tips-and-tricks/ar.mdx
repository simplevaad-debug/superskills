معظم المطورين يستخدمون 20% من قدرات Claude Code. الـ80% الأخرى موجودة هناك، غير مستخدمة — تنتظر أن تجعلك أسرع بشكل ملحوظ. هذه الـ25 نصيحة هي ما يميز المستخدمين العاديين عن المستخدمين المتقدمين. اعمل عليها مرة واحدة ولن تعود إلى الوراء.

## الإعداد

### 1. هيّئ CLAUDE.md لكل مشروع

`CLAUDE.md` هو أول ملف يقرأه Claude في أي مشروع. استخدمه لتعريف مجموعة أدواتك واتفاقياتك وتفضيلاتك — حتى لا تضطر لتكرار نفسك.

```markdown
# المشروع: E-commerce API
Stack: Node.js, TypeScript, PostgreSQL, Prisma, Express
Test runner: Vitest
Style: Prettier مع 2-space indent، علامات اقتباس مفردة
الاتفاقيات:
- استخدم async/await، ليس callbacks أبداً
- جميع مسارات API تحتاج تحققاً من المدخلات عبر Zod
- الأخطاء تمر عبر معالج الأخطاء المركزي
```

مع `CLAUDE.md` جيد، يعرف Claude اتفاقياتك من الرسالة الأولى في كل جلسة.

### 2. استخدم الذاكرة للاحتفاظ بالسياق

نظام ذاكرة Claude يتذكر الأشياء عبر الجلسات. أخبره الأشياء المهمة مرة واحدة:

```
/memory "نستخدم Railway للنشر. الخلفية في api.myapp.com.
الواجهة الأمامية في myapp.com. التجهيز في staging.myapp.com."
```

الآن لن تضطر لشرح إعداد النشر الخاص بك مرة أخرى.

### 3. عيّن شخصية افتراضية في CLAUDE.md

أضف هذا إلى `CLAUDE.md` الخاص بك:

```markdown
عندما لا أحدد غير ذلك، تصرف كمهندس TypeScript أول
يُقدّر البساطة والصحة وقابلية الصيانة على الذكاء.
```

هذا يشكل كل استجابة دون أن تضطر للسؤال.

### 4. أنشئ أوامر Slash خاصة بالمشروع

حدد أوامر slash مخصصة في دليل `.claude/commands/` بمشروعك:

```markdown
# .claude/commands/new-component.md
أنشئ مكوّن React جديد مع:
- واجهة props TypeScript
- تصدير افتراضي
- ملف اختبار وحدة أساسي
- قصة Storybook

اسم المكوّن: $ARGUMENTS
```

الآن `/new-component Button` ينشئ جميع الملفات الأربعة.

### 5. استخدم .claudeignore للحفاظ على السياق نظيفاً

مثل `.gitignore`، `.claudeignore` يخبر Claude الملفات التي يتخطاها:

```
node_modules/
dist/
.next/
*.generated.ts
coverage/
```

سياق أصغر = استجابات أسرع = إجابات أكثر تركيزاً.

## التنقل

### 6. أتقن مراجع @ للملفات

بدلاً من "الملف في src/components/auth/LoginForm.tsx"، اكتب `@src/components/auth/LoginForm.tsx`. يحصل Claude على محتوى الملف الدقيق دون نسخ ولصق.

سلسل مراجع متعددة: "قارن `@src/auth/v1.ts` بـ `@src/auth/v2.ts` واشرح الاختلافات."

### 7. استخدم # للإشارة إلى الذكريات المضمنة

إذا حفظت ذكريات، أشر إليها بـ`#`:

```
أضف تحديد المعدل لنقاط نهاية API كما هو محدد في #deployment-config
```

### 8. تعلم اختصارات لوحة المفاتيح الأساسية

- `Ctrl+K` (أو `Cmd+K`) — امسح السياق، ابدأ من جديد
- `Ctrl+R` — البحث في سجل المحادثة
- `Ctrl+L` — انتقل إلى آخر رسالة
- `Tab` — قبول الإكمال التلقائي في المدخلات
- `↑/↓` — التنقل في سجل الرسائل

### 9. استخدم أوامر Slash للمهام المتكررة

أوامر slash المدمجة التي يجب معرفتها:
- `/review` — وضع مراجعة الكود
- `/test` — توليد اختبارات للملف الحالي
- `/explain` — شرح الكود المحدد
- `/refactor` — إعادة هيكلة مع وصف الهدف
- `/commit` — توليد رسالة إيداع من التغييرات المُرحَّلة

### 10. ثبّت الملفات المهمة للجلسات الطويلة

في الجلسات الطويلة، استخدم مراجع `@file` للحفاظ على الملفات الحرجة في السياق:

```
بالنظر إلى @src/components/Checkout.tsx و@src/types/checkout.ts
و@src/tests/checkout.test.ts — ما أفضل طريقة لإضافة ميزة رمز الخصم؟
```

## سير العمل

### 11. استخدم Hooks للإجراءات التلقائية

خطافات Claude Code تعمل تلقائياً على الأحداث. الأكثر فائدة:

```json
{
  "PostToolUse": [
    {
      "matcher": "Edit",
      "hooks": [{ "type": "command", "command": "npx prettier --write $CLAUDE_FILE_PATHS" }]
    }
  ]
}
```

الآن كل ملف يحرره Claude يُنسَّق تلقائياً.

### 12. شغّل Agents متوازية للمهام المستقلة

عندما تكون لديك مهام مستقلة متعددة، لا تشغلها بالتسلسل. استخدم وكلاء فرعيين:

```
أطلق ثلاثة agents متوازية:
1. العامل 1: اكتب اختبارات وحدة لـ src/services/auth.ts
2. العامل 2: اكتب اختبارات وحدة لـ src/services/payment.ts
3. العامل 3: اكتب اختبارات وحدة لـ src/services/email.ts
أبلّغ عند الانتهاء من جميعها.
```

ثلاث مهام تُنجز في وقت واحد — أسرع 3 مرات من التسلسل.

### 13. استخدم Git Worktrees لتطوير الميزات المتوازي

Worktrees تتيح لك الحصول على فروع متعددة محوّلة في وقت واحد:

```bash
git worktree add ../feature-payments feature/payments
git worktree add ../feature-auth feature/auth
```

شغّل جلسة Claude واحدة في كل دليل — سياق مستقل بدون تداخل.

### 14. أنشئ قوالب Prompt قابلة لإعادة الاستخدام

احفظ أفضل مطالباتك كقوالب في `.claude/templates/`:

```markdown
# .claude/templates/api-endpoint.md
أنشئ نقطة نهاية API كاملة لـ[المورد]:
1. تعريف المسار مع طريقة HTTP المناسبة
2. التحقق من المدخلات بـ Zod
3. دالة طبقة الخدمة
4. استعلام قاعدة البيانات (Prisma)
5. معالجة الأخطاء
6. اختبار وحدة
7. اختبار تكامل
```

### 15. استخدم ميزة التراجع بحرية

يمتلك Claude Code نظام تراجع — إذا سار تغيير بشكل خاطئ، يمكنك التراجع عنه دون الرجوع يدوياً. استخدم هذا لجعل Claude أكثر جرأة في إعادة الهيكلة: "جرب إعادة الهيكلة الأكثر عدوانية — يمكننا دائماً التراجع."

## المهارات

### 16. اجمع المهارات للمهام المعقدة

المهارات تتراكم. اجمعها للخبرة المركبة:

```
مع تفعيل typescript-pro وreact-expert، أعد هيكلة مكوّن
جلب البيانات هذا لاستخدام Server Components وgenerics TypeScript المناسبة.
```

مهارتان نشطتان = خبير TypeScript + خبير React في استجابة واحدة.

### 17. فعّل المهارات بالسياق، وليس الأوامر

معظم المهارات تُفعَّل تلقائياً من السياق. لا تحتاج لكتابة "use typescript-pro skill." فقط:
- اعمل في ملف `.tsx` ← يتفعل `react-expert`
- اسأل عن Docker ← يتفعل `devops-engineer`
- اذكر خطأً ← يتفعل `debugging-wizard`

### 18. استخدم مشغلات مهارات مخصصة

يمكنك تعريف عبارات تشغيل مخصصة في CLAUDE.md:

```markdown
عندما أقول "وضع TypeScript الكامل"، فعّل typescript-pro وافرض
الوضع الصارم، لا `any`، generics مناسبة في كل مكان.
```

### 19. سلسل المهارات لمراجعة الكود + الإصلاح

مراجعة وإصلاح في مطالبة واحدة:

```
مع تفعيل code-reviewer، راجع هذا PR ثم، لكل مشكلة وُجدت،
اكتب فوراً الكود المصحح.
```

وضع المراجعة + وضع الإصلاح في أمر واحد.

### 20. أنشئ مهارات خاصة بالمجال لفريقك

الفرق يمكنها إنشاء مهارات مخصصة لمجموعتها التقنية المحددة:

```markdown
# .claude/skills/our-stack.md
أنت خبير في مجموعة تقنيات [الشركة]:
- Next.js 15 App Router
- Prisma مع PostgreSQL
- Clerk للمصادقة
- Vercel للنشر
اتبع دائماً اتفاقيتنا في [...]
```

## متقدم

### 21. استخدم خوادم MCP للبيانات الخارجية

خوادم Model Context Protocol (MCP) تتيح لـClaude الوصول المباشر لمصادر البيانات الخارجية:

```json
{
  "servers": {
    "postgres": {
      "command": "npx @modelcontextprotocol/server-postgres",
      "env": { "DATABASE_URL": "${DATABASE_URL}" }
    }
  }
}
```

الآن يمكن لـClaude الاستعلام عن مخطط قاعدة بياناتك الفعلية وكتابة استعلامات دقيقة.

### 22. ابنِ سير عمل الوكلاء الفرعيين للمهام الكبيرة

للمهام الكبيرة جداً لنافذة سياق واحدة، قسّمها إلى سلاسل وكيل فرعي:

```
المهمة: انقل قاعدة الكود بالكامل من JavaScript إلى TypeScript.
المرحلة 1: حلّل جميع الملفات وأنشئ خطة ترحيل
المرحلة 2: (لكل ملف) حوّل إلى TypeScript وأضف الأنواع
المرحلة 3: شغّل مجموعة الاختبارات الكاملة وأصلح أخطاء الأنواع
المرحلة 4: حدّث التوثيق
```

كل مرحلة هي استدعاء عامل منفصل — مخرجات كل منها تُغذّي التالية.

### 23. استخدم --continue للمهام طويلة المدى

عندما تُقطع مهمة في المنتصف (حد السياق)، استخدم `--continue` للاستئناف من المكان الذي توقف فيه بالضبط.

### 24. مرّر السياق الخارجي إلى Claude

تكامل Shell يعني أنك تستطيع تمرير بيانات النظام الحقيقية مباشرة:

```bash
# مراجعة السجلات الفعلية الجارية
docker logs my-app | claude "حدد أنماط الأخطاء في هذه السجلات"

# تحليل أداء استعلامات قاعدة البيانات الحقيقية
psql -c "EXPLAIN ANALYZE SELECT ..." | claude "اشرح خطة الاستعلام هذه واقترح تحسينات"

# مراجعة فروق PR
gh pr diff 142 | claude "راجع هذا PR لمشاكل الأمان"
```

بيانات حقيقية + Claude = تحليل أفضل بكثير من الأوصاف.

### 25. قِس سير عملك وحسّنه

تتبع أنماط المطالبة التي تعطيك أفضل النتائج. احتفظ بملف `prompts.md` شخصي:

```markdown
# أفضل مطالباتي

## مراجعة الكود
"راجع هذا الكود كمهندس أمان قرأ للتو OWASP Top 10.
علّم كل مشكلة حسب ترتيب الخطورة."

## قرار المعمارية
"أحتاج للاختيار بين [الخيار A] و[الخيار B].
افترض أنني أبني هذا لنطاق 10x من اليوم.
أعطني ثلاث حجج لكل جانب، ثم توصيتك."

## تصحيح الأخطاء
"لدي هذا الخطأ: [وصف]. إليك الكود: [الكود].
ضع ثلاث فرضيات حول السبب الجذري، مرتبة حسب الاحتمالية."
```

المطالبات التي تعمل تصبح أدوات قابلة لإعادة الاستخدام.
