Claude Code הוא כלי עוצמתי בפני עצמו. אבל skills הם מה שהופך אותו ממסייע מוכשר למומחה תחום שמבין את ה-stack שלך, עוקב אחרי המוסכמות שלך, ומצליח לפגוע בול בפעם הראשונה. אחרי שיחה עם מאות מפתחים, זיהינו את 10 ה-skills שמספקים את ההשפעה הגדולה ביותר — ללא קשר למה שאתה בונה.

הנה הם.

---

## 1. `react-expert`

**מה הוא עושה:** מעניק ל-Claude ידע עמוק בדפוסי React 18+ — Server Components, Suspense, קומפוזיציית hooks, תכונות concurrent, והמעבר מ-class components.

**מתי הוא מופעל:** עבודה עם קבצי `.tsx`/`.jsx`, יבוא מ-`react`, או אזכור React בפרומפט שלך.

**דוגמת שימוש:** אתה מבקש מ-Claude לבנות קומפוננטת data-fetching. בלי ה-skill, אולי תקבל דפוס `useEffect` שעובד אבל לא אופטימלי. עם `react-expert`, תקבל Server Component עם Suspense boundaries נכונים, מצבי loading וטיפול בשגיאות — React כמו שצריך להשתמש בו ב-2026.

---

## 2. `typescript-pro`

**מה הוא עושה:** אוכף פרקטיקות TypeScript מחמירות: אילוצי generic נכונים, discriminated unions, branded types, `noImplicitAny`, pattern matching ממצה, והימנעות מ-`any` בכל מחיר.

**מתי הוא מופעל:** קבצי TypeScript, משימות `tsconfig.json`, או שאלות הקשורות לטיפוסים.

**דוגמת שימוש:** אתה מבקש מ-Claude לטפל בתגובת API. בלי ה-skill, אולי תקבל `any` או interface רופף. עם `typescript-pro`, תקבל טיפוס narrowed כראוי עם ולידציה בזמן ריצה באמצעות Zod, branded ID type, ו-discriminated union למצבי success/error.

---

## 3. `test-master`

**מה הוא עושה:** מיישם עקרונות Test-Driven Development, כותב בדיקות משמעותיות (לא רק כאלה שמנפחות coverage), משתמש באסטרטגיות mocking נכונות, ומארגן test suites לתחזוקה לטווח ארוך.

**מתי הוא מופעל:** כתיבת קבצי test, אזכורים של Vitest/Jest/Playwright, או בקשה מ-Claude "להוסיף tests."

**דוגמת שימוש:** אתה מבקש מ-Claude לכתוב tests לתהליך הרשמת משתמש. עם `test-master`, תקבל unit tests ללוגיקת העסקים, integration tests לשכבת ה-API, ו-E2E tests לנסיעת המשתמש המלאה — כל אחד עם הרמה הנכונה של mocking ו-assertions משמעותיים.

---

## 4. `devops-engineer`

**מה הוא עושה:** הופך את Claude למומחה ב-infrastructure as code: Dockerfiles, Docker Compose, GitHub Actions, CI/CD pipelines, Kubernetes manifests, Terraform ואסטרטגיות deployment.

**מתי הוא מופעל:** עבודה עם `Dockerfile`, `.github/workflows/`, `docker-compose.yml`, או אזכור deployment.

**דוגמת שימוש:** אתה מבקש מ-Claude להגדיר CI/CD pipeline לאפליקציית Node.js. עם `devops-engineer`, תקבל GitHub Actions workflow מובנה היטב עם caching נכון, שלבי build/test/deploy נפרדים, טיפול במשתני סביבה ואסטרטגיית rollback — לא רק YAML של 10 שורות שעובד טכנית.

---

## 5. `python-pro`

**מה הוא עושה:** אוכף דפוסי Python מודרניים: type hints, dataclasses, async/await, טיפול נכון בשגיאות עם exceptions מותאמות, ניהול virtual environment ואידיומים Pythonic.

**מתי הוא מופעל:** קבצי `.py`, `requirements.txt`, `pyproject.toml`, או שאלות הקשורות ל-Python.

**דוגמת שימוש:** אתה מבקש מ-Claude לכתוב סקריפט עיבוד נתונים. עם `python-pro`, תקבל פונקציות עם טיפוסים נכונים, context managers לניקוי משאבים, `__main__` guard, logging במקום print statements, והפרדה נקייה בין פונקציות טהורות ל-I/O — קוד שמפתח Python בכיר יהיה גאה בו.

---

## 6. `security-reviewer`

**מה הוא עושה:** מיישם חשיבה ביטחונית לכל קוד: מניעת SQL injection, הפחתת XSS, דפוסי אימות נכונים, ניהול secrets, הגדרת CORS, ולידציית input ומודעות ל-OWASP Top 10.

**מתי הוא מופעל:** קוד הקשור לאימות, API routes, שאילתות בסיס נתונים, או בקשה מפורשת לביקורת אבטחה.

**דוגמת שימוש:** אתה מבקש מ-Claude להוסיף API endpoint. עם `security-reviewer`, Claude מוסיף אוטומטית ולידציית input עם ספריית schema, בודק שהמשתמש מאומת ומורשה (לא רק מאומת), מחטא כל נתון שנכנס לשאילתות, ומסמן בעיות פוטנציאליות בגוף התגובה.

---

## 7. `debugging-wizard`

**מה הוא עושה:** מעניק ל-Claude מתודולוגיית debugging מובנית: גיבוש היפותזות, סילוק שיטתי, ניתוח שורש הבעיה, והמשמעת לתקן את הבעיה האמיתית ולא את הסימפטום.

**מתי הוא מופעל:** תיאורי bugs, הודעות שגיאה, פרומפטים של "זה לא עובד", או stack traces.

**דוגמת שימוש:** אתה מדביק הודעת שגיאה קריפטית. עם `debugging-wizard`, Claude לא פשוט מחפש את השגיאה בגוגל. הוא עובר דרך היפותזה: מה הסיבה הסבירה, מה יאשר אותה, איך לבודד את המשתנה, מה הפתרון צריך להיות — ומציע גם איך למנוע את הבעיה בעתיד.

---

## 8. `nextjs-developer`

**מה הוא עושה:** מומחיות עמוקה ב-Next.js App Router: Server לעומת Client Components, דפוסי RSC, route handlers, middleware, `generateMetadata`, streaming, אופטימיזציית תמונות, ו-nuances של גבולות `use client`.

**מתי הוא מופעל:** קבצי פרויקט Next.js, תיקיות App Router, או משימות `next.config.js`.

**דוגמת שימוש:** אתה מבקש מ-Claude להוסיף תמונות OG דינמיות לבלוג שלך. עם `nextjs-developer`, תקבל קובץ `opengraph-image.tsx` נכון המשתמש ב-API ליצירת תמונות של Next.js, הגדרת metadata נכונה, כותרות caching והגדרת edge runtime — לא פתרון גנרי ששובר את ה-build.

---

## 9. `api-designer`

**מה הוא עושה:** מיישם best practices של REST ו-GraphQL: מתן שמות למשאבים נכון, קודי HTTP status, אסטרטגיית versioning, דפוסי pagination, צורות תגובת שגיאה, זרימות אימות ומוסכמות תיעוד API.

**מתי הוא מופעל:** בניית routes/endpoints, עיצוב schemas, או שאלות על מבנה API.

**דוגמת שימוש:** אתה מבקש מ-Claude לעצב REST API לאפליקציית ניהול משימות. עם `api-designer`, תקבל היררכיית משאבים עקבית, מתן שמות עקבי (`/tasks` ולא `/getTasks`), שימוש נכון ב-HTTP methods, פורמט תגובת שגיאה מתוקנן ואסטרטגיית versioning שלא תשבור clients כשאתה מבצע איטרציות.

---

## 10. `prompt-engineer`

**מה הוא עושה:** הופך את Claude למומחה בכתיבת prompts למערכות AI אחרות — פלטים מובנים, chain-of-thought, few-shot examples, עיצוב system prompt וטיפול ב-edge cases בתכונות מבוססות LLM.

**מתי הוא מופעל:** בניית תכונות מופעלות ב-AI, עבודה עם קריאות API של OpenAI/Anthropic, או בקשה מ-Claude לעזור בכתיבת prompts.

**דוגמת שימוש:** אתה מוסיף תכונת ביקורת מופעלת ב-AI לאפליקציה שלך. עם `prompt-engineer`, Claude מעצב system prompt עם אילוצי פלט ברורים, few-shot examples שמטפלות ב-edge cases, אסטרטגיית fallback לתגובות שגויות והצעות לאופטימיזציית עלויות — לא רק `"You are a helpful assistant. Review this code."`.
