Claude Code قوي في حد ذاته. لكن المهارات هي ما يحوّله من مساعد قادر إلى خبير في المجال يفهم مكدسك التقني، ويتبع اتفاقياتك، ويُصيب الهدف في المرة الأولى. بعد التحدث مع مئات المطورين، حددنا أكثر 10 مهارات تأثيراً — بغض النظر عما تبنيه.

إليك هذه المهارات.

---

## 1. `react-expert`

**ما تفعله:** تمنح Claude معرفة عميقة بأنماط React 18+ — Server Components وSuspense وتأليف hooks والميزات المتزامنة والانتقال بعيداً عن class components.

**متى تُفعَّل:** العمل مع ملفات `.tsx`/`.jsx`، الاستيراد من `react`، أو ذكر React في موجّهك.

**مثال على حالة الاستخدام:** تطلب من Claude بناء مكوّن لجلب البيانات. بدون المهارة، قد تحصل على نمط `useEffect` يعمل لكنه غير مثالي. مع `react-expert`، تحصل على Server Component مع حدود Suspense مناسبة، وحالات التحميل، ومعالجة الأخطاء — React كما ينبغي استخدامه في عام 2026.

## 2. `typescript-pro`

**ما تفعله:** تفرض ممارسات TypeScript الصارمة: قيود generic مناسبة، discriminated unions، branded types، `noImplicitAny`، مطابقة الأنماط الاستنفادية، وتجنب `any` بكل الوسائل.

**متى تُفعَّل:** ملفات TypeScript، مهام `tsconfig.json`، أو أسئلة متعلقة بالأنواع.

**مثال على حالة الاستخدام:** تطلب من Claude كتابة نوع لاستجابة API. بدون هذه المهارة، قد تحصل على `any` أو واجهة مفككة. مع `typescript-pro`، تحصل على نوع مُضيَّق بشكل صحيح مع التحقق من الصحة في وقت التشغيل باستخدام Zod، ونوع ID مُعلَّم، وunion مُميَّزة لحالات النجاح/الخطأ.

## 3. `test-master`

**ما تفعله:** تُطبّق مبادئ Test-Driven Development، وتكتب اختبارات ذات معنى (وليس فقط لتضخيم التغطية)، وتستخدم استراتيجيات mocking مناسبة، وتُهيكل مجموعات الاختبار للصيانة على المدى البعيد.

**متى تُفعَّل:** كتابة ملفات الاختبار، ذكر Vitest/Jest/Playwright، أو طلب من Claude "إضافة اختبارات."

**مثال على حالة الاستخدام:** تطلب من Claude كتابة اختبارات لتدفق تسجيل المستخدم. مع `test-master`، تحصل على اختبارات وحدة لمنطق الأعمال، واختبارات تكامل لطبقة API، واختبارات E2E لرحلة المستخدم الكاملة — كل منها بالمستوى الصحيح من التوهيم والتوكيدات ذات المعنى.

## 4. `devops-engineer`

**ما تفعله:** تجعل Claude خبيراً في البنية التحتية كرمز: Dockerfiles، وDocker Compose، وGitHub Actions، وCI/CD pipelines، وKubernetes manifests، وTerraform، واستراتيجيات النشر.

**متى تُفعَّل:** العمل مع `Dockerfile`، `.github/workflows/`، `docker-compose.yml`، أو ذكر النشر.

**مثال على حالة الاستخدام:** تطلب من Claude إعداد CI/CD pipeline لتطبيق Node.js. مع `devops-engineer`، تحصل على سير عمل GitHub Actions منظم جيداً مع التخزين المؤقت الصحيح، ومراحل build/test/deploy منفصلة، ومعالجة متغيرات البيئة، واستراتيجية التراجع — وليس مجرد YAML من 10 سطور يعمل من الناحية التقنية.

## 5. `python-pro`

**ما تفعله:** تفرض أنماط Python الحديثة: تلميحات النوع، وdataclasses، وasync/await، ومعالجة الأخطاء الصحيحة مع استثناءات مخصصة، وإدارة البيئة الافتراضية، والأساليب Pythonic.

**متى تُفعَّل:** ملفات `.py`، `requirements.txt`، `pyproject.toml`، أو أسئلة متعلقة بـ Python.

**مثال على حالة الاستخدام:** تطلب من Claude كتابة نص برمجي لمعالجة البيانات. مع `python-pro`، تحصل على دوال ذات أنواع صحيحة، ومديري سياق لتنظيف الموارد، وحارس `__main__`، وتسجيل بدلاً من عبارات print، وفصلاً نظيفاً بين الدوال النقية والـ I/O — كود يفخر به مطور Python أول.

## 6. `security-reviewer`

**ما تفعله:** تُطبّق التفكير الأمني على كل قطعة من الكود: منع SQL injection، وتخفيف XSS، وأنماط المصادقة الصحيحة، وإدارة الأسرار، وإعداد CORS، والتحقق من صحة المدخلات، والوعي بـ OWASP Top 10.

**متى تُفعَّل:** الكود المتعلق بالمصادقة، ومسارات API، واستعلامات قواعد البيانات، أو طلب مراجعة أمنية صريحة.

**مثال على حالة الاستخدام:** تطلب من Claude إضافة نقطة نهاية API. مع `security-reviewer`، يضيف Claude تلقائياً التحقق من صحة المدخلات مع مكتبة schema، ويتحقق من مصادقة المستخدم وتفويضه (وليس المصادقة فحسب)، ويُعقّم أي بيانات تدخل الاستعلامات، ويُعلّم أي مشاكل محتملة في جسم الاستجابة.

## 7. `debugging-wizard`

**ما تفعله:** تمنح Claude منهجية debugging منظمة: صياغة الفرضيات، والاستبعاد المنهجي، وتحليل السبب الجذري، والانضباط لإصلاح المشكلة الفعلية لا الأعراض.

**متى تُفعَّل:** أوصاف الأخطاء، ورسائل الخطأ، وموجّهات "لا يعمل"، أو تتبعات المكدس.

**مثال على حالة الاستخدام:** تلصق رسالة خطأ غامضة. مع `debugging-wizard`، لا يبحث Claude فقط عن الخطأ. بل يمر عبر فرضية: ما هو السبب المحتمل، وما الذي سيؤكد ذلك، وكيفية عزل المتغير، وما ينبغي أن يكون الإصلاح — بالإضافة إلى اقتراح كيفية منع المشكلة مستقبلاً.

## 8. `nextjs-developer`

**ما تفعله:** خبرة عميقة في Next.js App Router: مكونات Server مقابل Client، وأنماط RSC، ومعالجات المسار، والـ middleware، و`generateMetadata`، والبث، وتحسين الصور، ودقائق حدود `use client`.

**متى تُفعَّل:** ملفات مشروع Next.js، ومجلدات App Router، أو مهام `next.config.js`.

**مثال على حالة الاستخدام:** تطلب من Claude إضافة صور OG ديناميكية لمدونتك. مع `nextjs-developer`، تحصل على ملف `opengraph-image.tsx` صحيح يستخدم واجهة برمجة تطبيقات توليد الصور في Next.js، وإعداد metadata صحيح، وترويسات التخزين المؤقت، وإعداد edge runtime — وليس حلاً عاماً يكسر عملية البناء.

## 9. `api-designer`

**ما تفعله:** تُطبّق أفضل ممارسات REST وGraphQL: تسمية الموارد بشكل صحيح، ورموز حالة HTTP، واستراتيجية الإصدار، وأنماط pagination، وأشكال استجابة الخطأ، وتدفقات المصادقة، واتفاقيات توثيق API.

**متى تُفعَّل:** بناء المسارات/نقاط النهاية، وتصميم المخططات، أو السؤال عن بنية API.

**مثال على حالة الاستخدام:** تطلب من Claude تصميم REST API لتطبيق إدارة المهام. مع `api-designer`، تحصل على تسلسل هرمي متسق للموارد، وتسمية متسقة (`/tasks` وليس `/getTasks`)، واستخدام صحيح لأساليب HTTP، وتنسيق استجابة خطأ موحد، واستراتيجية إصدار لن تكسر العملاء عند التكرار.

## 10. `prompt-engineer`

**ما تفعله:** تحوّل Claude إلى خبير في كتابة الموجّهات لأنظمة AI أخرى — المخرجات المنظمة، وchain-of-thought، وأمثلة few-shot، وتصميم system prompt، ومعالجة الحالات الحدية في الميزات المستندة إلى LLM.

**متى تُفعَّل:** بناء ميزات مدعومة بالذكاء الاصطناعي، والعمل مع استدعاءات API لـ OpenAI/Anthropic، أو طلب مساعدة Claude في كتابة الموجّهات.

**مثال على حالة الاستخدام:** تضيف ميزة مراجعة مدعومة بالذكاء الاصطناعي إلى تطبيقك. مع `prompt-engineer`، يصمم Claude system prompt بقيود إخراج واضحة، وأمثلة few-shot تتعامل مع الحالات الحدية، واستراتيجية احتياطية للاستجابات المشوهة، واقتراحات تحسين التكلفة — وليس فقط `"You are a helpful assistant. Review this code."`.
