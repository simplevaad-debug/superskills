إذا كنت تستخدم Claude Code — واجهة سطر الأوامر من Anthropic للتطوير بمساعدة الذكاء الاصطناعي — فمن المحتمل أنك لاحظت أنه قوي جداً. لكن ماذا لو كان بإمكانك جعله أفضل؟ هذا بالضبط ما تفعله **مهارات Claude Code**.

## ما هي المهارة؟

المهارة هي ملف prompt منظم يمنح Claude Code **خبرة متخصصة في مجال معين**. فكّر فيها كدليل عمل: بدلاً من شرح معايير الكود والأنماط المفضلة في كل جلسة، تكتبها مرة واحدة كمهارة، ويتبعها Claude تلقائياً.

كل ملف مهارة موجود في `~/.claude/skills/` ويتم تفعيله عندما تتحقق شروط التفعيل. على سبيل المثال، مهارة `react-expert` تُفعّل عند العمل مع ملفات `.tsx` وذكر React — فيتبع Claude أفضل ممارسات React 18+.

## كيف تعمل المهارات؟

المهارات هي ملفات markdown بسيطة مع بيانات وصفية:

```yaml
---
name: react-expert
description: Use when building React 18+ applications
---

## Rules

- Always use functional components with hooks
- Prefer Server Components by default
- Use Suspense boundaries for async operations
```

عندما يكتشف Claude Code سياقاً مطابقاً، يحمّل المهارة ويتبع تعليماتها. لا حاجة لمفاتيح API، لا إعدادات، لا سكريبتات تثبيت — مجرد ملفات في مجلد.

### دورة حياة المهارة

1. **الاكتشاف** — Claude Code يمسح مجلد المهارات عند بدء التشغيل
2. **المطابقة** — عند إعطاء مهمة، يتحقق Claude من المهارات ذات الصلة
3. **التفعيل** — يتم تحميل المهارات المطابقة في سياق Claude
4. **التنفيذ** — يتبع Claude تعليمات المهارة أثناء العمل

## لماذا يجب استخدام المهارات؟

### 1. الاتساق عبر الجلسات

بدون مهارات، عليك إعادة شرح تفضيلاتك في كل مرة. المهارات تُشفّر هذه التفضيلات مرة واحدة.

### 2. مخرجات بمستوى خبير

المهارة الجيدة لا تحدد التفضيلات فقط — بل تعلّم Claude أفضل الممارسات.

### 3. السرعة

بدلاً من تكرار مخرجات Claude، المهارات تحصل على النتيجة الصحيحة من المحاولة الأولى.

### 4. محاذاة الفريق

شارك المهارات مع فريقك. الجميع يحصل على نفس المعايير ونفس الجودة.
