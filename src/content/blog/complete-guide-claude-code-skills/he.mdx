## מה הם סקילי Claude Code?

סקיל Claude Code הוא קובץ `.md` שמתקינים ב-`~/.claude/skills/`. כל סקיל מכיל הוראות מפורטות ומובנות שמנחות את התנהגות Claude בתחום ספציפי.

המושג המרכזי: **סקילים אינם משנים את מה ש-Claude יכול לעשות, הם משנים כיצד Claude מיישם את מה שהוא יודע.**

ל-Claude כבר יש ידע בתחומי אבטחה, בסיסי נתונים, DevOps ופיתוח frontend. אבל ללא הקשר תחום, הוא מיישם את הידע הזה בצורה גנרית. סקיל אומר ל-Claude: "בתחום זה, אלו הם הסטנדרטים שחשובים, אלו התבניות שיש לעקוב אחריהן, ואלו הטעויות שיש להימנע מהן."

ההבדל באיכות הפלט משמעותי. בקשו מ-Claude לכתוב זרימת אימות ללא סקיל ותקבלו משהו שעובד. בקשו עם סקיל `secure-code-guardian` פעיל ותקבלו משהו מוכן לייצור — עם rate limiting, ניהול session, טיפול נכון בשגיאות ותבניות תואמות OWASP.

### כיצד סקילים נטענים

סקילים הם קבצי Markdown פשוטים. Claude Code קורא קבצים ב-`~/.claude/skills/` אוטומטית כשהוא מתחיל סשן. אפשר גם לטעון סקילים ספציפיים במפורש על ידי הפנה אליהם בשיחה.

סקילים מרובים מצטברים — אפשר שיהיו `frontend-developer`, `design-system-architect` ו-`testing-engineer` כולם פעילים באותו סשן. Claude מסנתז הנחיות מכל הסקילים שנטענו.

### בניית לעומת קניית סקילים

אפשר לכתוב סקילים משלכם — סקיל הוא רק קובץ Markdown עם הוראות. האתגר הוא כתיבת סקילים שאפקטיביים בפועל: מכסים את עומק הידע הנכון בתחום, מובנים בדרכים ש-Claude מיישם באמינות, ומקיפים מספיק לטיפול במורכבות העולם האמיתי.
